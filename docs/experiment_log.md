# Experiment Log

Running documentation of experiment results, issues encountered, and lessons learned. Intended as a reference for developers continuing this project.

## Experiment 1: V1 Schema (Single-Ticker Sentiment)

**Date**: June 7, 2026
**Objective**: Fine-tune Qwen3-8B to produce structured sentiment signals for a primary ticker from quantum computing news.

### Setup
- Base model: Qwen/Qwen3-8B
- Fine-tuning: QLoRA, rank 64, 4 epochs
- Training data: 199 examples generated by qwen3-max (teacher)
- Schema: Single primary ticker with sentiment, event_type, confidence, technical_translation
- Hardware: Modal A100-SXM4-40GB
- Training time: ~4 minutes

### Results
- Final training loss: 1.96
- Test accuracy: 3/3 (100% on held-out test articles)
- JSON compliance: 100% on test set
- Deployed to HF Space successfully

### Issues
- 60+ second latency on HF Space (ZeroGPU cold start + thinking mode enabled by default)
- Output felt "surface level" per user feedback (just bullish/bearish without actionable quant detail)
- Confidence scores felt arbitrary (no grounding)

### Lessons Learned
- Single-ticker sentiment is too simple for quant integration
- Need cross-sectional signals (all tickers per article) for Alphalens evaluation
- Disable thinking mode by default for fast inference; offer as toggle

## Experiment 2: V2 Schema (Cross-Sectional Signal Vector)

**Date**: June 8, 2026
**Objective**: Fine-tune Qwen3-8B to produce signal vectors across all 9 quantum tickers for every article.

### Setup
- Base model: Qwen/Qwen3-8B
- Fine-tuning: QLoRA, rank 64, 4 epochs
- Training data: 199 examples generated by qwen3-max (V2 signal vector schema)
- Schema: Full signal vector (9 tickers with score + reasoning), event_type, time_horizon, signal_decay, information_novelty, technical_translation, signal_rationale
- Hardware: Modal A100-SXM4-40GB
- Training time: ~7 minutes (longer due to more output tokens per example)

### Results
- Final training loss: 1.36 (improved from 1.96, more structure to learn)
- JSON compliance: ~79% on evaluation set (215/232 successful, 17 parse errors)
- Evaluation (215 predictions, 9063 observations):

| Metric | Value | Significance |
|--------|-------|-------------|
| IC at +1 day | +0.026 | Not significant (p=0.247) |
| IC at +2 days | -0.004 | Not significant |
| IC at +5 days | +0.050 | Significant at 5% (p=0.027) |
| IC at +10 days | +0.069 | Significant at 1% (p=0.003) |
| IC at +20 days | +0.016 | Not significant |
| Direction accuracy (+5d) | 56.8% | Above random (50%) |

### Key Findings
1. **Slow decay pattern confirmed**: Signal peaks at +10 days, consistent with hypothesis that quantum computing news has slow information diffusion
2. **News source works, arXiv does not**: IC for news = +0.053 (significant), IC for arXiv = -0.013 (not significant)
3. **HON (Quantinuum) strongest signal**: IC = +0.219 (p=0.001), likely because Quantinuum is a pure-play trapped-ion company where quantum news directly impacts the stock
4. **GOOGL negative IC**: Model struggles with diversified companies where quantum is <0.1% of revenue
5. **Government funding events strongest**: IC = +0.260 (p=0.027), predictable sector-wide effects

### Issues Encountered

#### Issue 1: Modal Timeout (Connection Drop)
- **Symptom**: `modal run` jobs die after ~20 minutes with "APP_STATE_STOPPED"
- **Root cause**: The sandbox's connection to Modal servers drops, killing the heartbeat, which stops the app
- **Fix**: Use `modal run --detach` and make the function fully self-contained (reads from volume, writes to volume)
- **Status**: Fixed

#### Issue 2: JSON Parse Errors (21% then 7%)
- **Symptom**: Model outputs malformed JSON for some articles
- **Root cause**: RSS-sourced articles contain HTML tags and Google News redirect URLs that confuse the model
- **Fix**: Strip HTML tags and URLs from article text before inference
- **Result**: Error rate dropped from 21% to 7%
- **Status**: Partially fixed (7% still failing, likely need more training data or constrained decoding)

#### Issue 3: Slow Inference (~58 seconds per article)
- **Symptom**: Expected 8-10 seconds per article, actual is 55-60 seconds
- **Root cause**: Not fully diagnosed. Likely combination of: (a) long input sequences for some articles, (b) model generating long output before producing JSON, (c) per-call overhead in Modal's container routing
- **Status**: Open. Need to analyze timing data once full run completes.

#### Issue 4: Temporal Leakage in Train/Eval Split
- **Symptom**: Training data (indices 0-199) contains articles from May-June 2026, while evaluation data (indices 200+) contains articles from Aug 2024 to May 2026
- **Root cause**: Articles file is sorted by date descending (newest first), and we used a simple index-based split
- **Impact**: Training on future data, evaluating on past data. This is inverted from proper walk-forward validation.
- **Mitigation**: The model learns text-to-JSON formatting (not price patterns), so the leakage is less severe than in traditional ML-on-prices. However, it's still methodologically incorrect.
- **Fix planned**: Re-sort chronologically, use date-based cutoff for next iteration
- **Status**: Acknowledged, fix planned for next iteration

#### Issue 5: Narrow Evaluation Window
- **Symptom**: Most evaluation predictions cluster in May 10-22, 2026 (12 calendar days)
- **Root cause**: RSS feeds heavily favor recent content; arXiv papers also skew recent
- **Impact**: IC results may not generalize to other market regimes
- **Fix planned**: Collect articles with broader date distribution for next iteration
- **Status**: Acknowledged as limitation

### Lessons Learned
- Signal vector schema is much more useful than single-ticker sentiment for quant evaluation
- Revenue exposure scaling works well (GOOGL scores stay near zero as intended)
- The model correctly identifies competitive dynamics (trapped-ion breakthrough = bearish for superconducting)
- 199 training examples is sufficient for basic formatting but insufficient for robust JSON compliance
- Modal's `--detach` mode is essential for long-running GPU jobs from ephemeral environments
- HTML cleaning is critical for RSS-sourced articles
